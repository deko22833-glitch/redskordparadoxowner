# Настройка для работы на GitHub сервере

## Автоматическая конфигурация

Код автоматически определяет окружение и настраивает подключение к серверу:

- **GitHub Pages** (`*.github.io`): Использует текущий домен как API URL
- **Локальная разработка** (`localhost`): Использует `http://localhost:3000`
- **Другие домены**: Использует текущий домен

## Если нужно указать явно

Если ваш бэкенд на другом сервере, укажите URL в `index.html` и `login.html`:

```html
<script>
    window.API_BASE_URL = 'https://your-backend-server.com';
</script>
```

## Проверка работы

1. Откройте консоль браузера (F12)
2. Проверьте, что нет ошибок подключения
3. Убедитесь, что Socket.IO подключается успешно

## Если что-то не работает

1. Проверьте, что сервер запущен и доступен
2. Проверьте CORS настройки в `server.js`
3. Убедитесь, что порт правильный (по умолчанию 3000)
4. Проверьте переменные окружения `ALLOWED_ORIGINS` на сервере

## Переменные окружения для сервера

Если используете GitHub Codespaces или другой хостинг:

```bash
PORT=3000
JWT_SECRET=your-secret-key-here
ALLOWED_ORIGINS=https://your-username.github.io,http://localhost:3000
```
